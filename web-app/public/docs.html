프로젝트 개요

냉장고의 실시간 온도를 측정하여 클라우드 DB에 저장하고, 웹앱(PWA)으로 어디서든 모니터링할 수 있는 IoT 시스템입니다.

시스템 아키텍처

[PT100 센서] → [MAX31865] →(SPI)→ [ESP32] →(Wi-Fi)→ [Firebase] → [웹앱/모바일]
구성 요소역할
PT100온도 센서 (냉장고 내부 설치)
MAX31865RTD-to-Digital 변환 모듈 (SPI 통신)
ESP32Wi-Fi 내장 마이크로컨트롤러
Firebase Firestore클라우드 NoSQL DB
웹앱 PWA모바일 브라우저에서 앱처럼 사용

조립 순서

  1. PT100 선 확인 - 2선식(선 2개) 또는 3선식(선 3개) 구분
  2. MAX31865 점퍼 설정 - 보드 뒷면 납땜 점퍼를 PT100 타입에 맞게 설정
    • 2선식: "2/4" 점퍼 + "2-3" 점퍼 둘 다 납땜
    • 3선식: "2-3" 점퍼만 끊기
  3. PT100 → MAX31865 - 스크류 터미널에 PT100 선 삽입 후 나사 조임
  4. ESP32 ↔ MAX31865 SPI 배선 - 위 배선표 참고
  5. 브레드보드 배치 - 부품 배치 후 점퍼선 연결
  6. 연결 확인 - 합선 없는지 체크
  7. USB 연결 테스트 - LED 점등 확인

납땜 없이 진행하기

헤더핀 미납땜 시 악어클립/듀폰 커넥터로 임시 연결 가능 (불안정). 권장: 저렴한 인두기 세트 구매 (1~2만원) 또는 메이커 스페이스 방문.

냉장고 설치 팁

  • PT100만 냉장고 내부에 넣기
  • 리드선은 문 패킹 아래쪽 틈새로 통과
  • ESP32 + MAX31865는 냉장고 외부에 배치
흔한 실수: VIN을 5V에 연결 (보드 손상), SDO↔SDI 핀 순서 뒤바뀜, PT100 나사 헐거움 (온도값 불안정)

컴파일 확인

빨간 글씨 "#pragma message: BT: forcing BR/EDR max sync conn eff to 1"은 정보성 메시지로, 에러가 아닙니다. 블루투스 관련 경고이며 프로젝트와 무관합니다.

정상 컴파일 결과: 프로그램 저장 공간 81%, 동적 메모리 15% 사용

Firebase 설정

왜 Firebase인가?

  • Gmail 계정만 있으면 즉시 사용 (AWS 가입 불필요)
  • 별도 서버 구축 불필요 (BaaS)
  • 실시간 데이터 동기화 내장
  • 무료 티어로 충분 (개인 프로젝트)

초기 설정

  1. Firebase 콘솔에서 프로젝트 생성
  2. Authentication → 익명 인증 활성화
  3. Firestore Database → 서울 리전, 테스트 모드로 생성
  4. Hosting → 웹앱 배포용

Firestore 데이터 구조

temperatures (컬렉션)
  └─ {docId} (문서)
      ├─ temperature: 4.5        (숫자)
      ├─ recorded_at: Timestamp  (시간)
      ├─ device_id: "fridge_01"  (문자열)
      └─ alert: false            (불리언)

metadata (컬렉션)
  └─ cleanup_last (문서)
      └─ timestamp: Timestamp    (시간)

messages (컬렉션)
  └─ {docId} (문서)
      ├─ text: "메시지 내용"       (문자열, 1~500자)
      ├─ uid: "abc123"            (익명 인증 UID)
      ├─ nickname: "관리자1"       (문자열, 1~20자)
      └─ created_at: Timestamp    (시간)

presence (컬렉션)
  └─ {uid} (문서)
      ├─ last_seen: Timestamp     (시간, 60초 하트비트)
      └─ uid: "abc123"            (익명 인증 UID)

typing (컬렉션)
  └─ {uid} (문서)
      ├─ last_typed: Timestamp    (시간, 3초 디바운스)
      └─ uid: "abc123"            (익명 인증 UID)

웹앱 가이드

기능 목록

  • 현재 온도 실시간 표시 (Firebase 리스너)

  • 최저 / 평균 / 최고 통계
  • 다중 디바이스 지원 (탭 전환)
  • 온도 알림 설정 (상한/하한, 브라우저 알림)
  • 차트 임계값 라인 표시
  • 라이트/다크 모드 전환
  • CSV 데이터 내보내기
  • PWA (홈 화면에 앱처럼 추가 가능)
  • 오프라인 캐싱 (Service Worker)
  • 한국어/영어 다국어 지원 (i18n)
  • 디바이스별 알림 임계값 설정
  • 알림 소리 (Web Audio API)
  • 화씨(°F) 단위 지원
  • 알림 이력 로그
  • 인증 실패 시 자동 재시도
  • 기록 목록 페이지네이션
  • 접근성(ARIA) 개선
  • 30일 이상 오래된 데이터 자동 정리
  • 익명 채팅 (Firestore 실시간, 플로팅 버튼 UI)
  • 채팅 읽지 않은 메시지 배지 (FAB 버튼)
  • 채팅 중복 전송 방지
  • 채팅 스마트 자동 스크롤
  • 채팅 날짜 구분선
  • ESC 키로 채팅 닫기
  • 채팅 전송 실패 피드백
  • 채팅 접속자 수 실시간 표시 (Firestore Presence)
  • 채팅 타이핑 인디케이터

트러블슈팅

ESP32 포트가 안 보여요

원인: CP2102N USB to UART Bridge 드라이버 미설치

해결:

  1. Silicon Labs 공식 드라이버 다운로드:
    https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers
  2. "CP210x Universal Windows Driver" 선택
  3. CP210xVCPInstaller_x64.exe 실행
  4. ESP32 USB 케이블 뽑았다가 다시 연결

컴파일 시 빨간 에러 메시지

메시지: #pragma message: BT: forcing BR/EDR max sync conn eff to 1

원인: ESP32 블루투스 라이브러리의 정보성 메시지 (경고)

해결: 무시해도 됩니다. 컴파일 성공입니다.

온도값이 불안정해요

확인 사항:

  • PT100 스크류 터미널 나사가 충분히 조여졌는지 확인
  • 2선/3선 점퍼 설정이 PT100 타입과 일치하는지 확인
  • SPI 배선 (특히 SDO↔SDI)이 바뀌지 않았는지 확인

웹앱에 데이터가 안 나와요

확인 사항:

  • Firebase 콘솔에서 Firestore에 데이터가 있는지 확인
  • Authentication → 익명 인증이 활성화되어 있는지 확인
  • 브라우저 콘솔(F12)에서 에러 메시지 확인
  • 테스트: cd test-firebase && npm run send로 가짜 데이터 전송 후 확인
